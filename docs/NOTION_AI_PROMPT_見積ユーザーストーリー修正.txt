# 見積・図面アップロード・図面自動拾い ユーザーストーリー・受け入れ基準の修正プロンプト（Notion AI用）

## 前提
「今Cursorに実装されているものが正」として、ユーザーストーリーと受け入れ基準を実装に合わせて修正する。

---

## 実装の現状（差分の要約）

### 図面アップロード・図面一覧・拾い導線
- **図面一覧は廃止済み**  
  アップロード後に図面を一覧表示するテーブル（図面名・登録日・ステータス・操作）は存在しない。図面はプロジェクトの `drawings` に登録されるが、一覧も検索も表示しない。
- **対象ページ指定は廃止済み**  
  「検索対象ページを指定する」ダイアログ・図面詳細内の対象ページ指定UIは削除済み。対象ページ未指定時は全ページで拾うロジックはコードに残るが、拾いを開始するUIがない。
- **自動拾いの「自動実行」は未実装**  
  アップロード直後に自動拾いが自動実行される処理はない。また**自動拾いを開始するボタンもない**（図面一覧廃止に伴い「自動拾い開始」ボタンを削除したため）。
- **結果・詳細・再実行・アーカイブ・削除の導線なし**  
  図面詳細ダイアログ・拾い結果ダイアログ・再実行ボタン等のコードは残っているが、これらを開くUI（一覧の「結果」「詳細」等）が存在しないため、画面からは利用できない。
- **拾い・LLM設定UIは廃止済み**  
  モデル・温度・誤差閾値の設定画面は削除済み。US-0503/0504の「UI上の設定画面は廃止」は実装と一致。

### 見積（EPIC-06）
- **拾い結果の出所**  
  「拾い結果から見積を作成」は**算出結果タブ**の「拾い結果から見積を作成」ボタンで実行される。拾い結果は**図面の自動拾いではなく「デモを読み込む」で投入した算出結果（extractedItems）**を利用。図面→自動拾い→結果→見積の導線はUI上存在しない。
- **見積一覧・編集・確定・取消・Excel**  
  見積一覧・状態/期間/キーワード絞り込み、見積編集（明細CRUD）、顧客・案件紐づけ、確定・取消・Excel出力は実装あり。取消理由は入力必須（保存時バリデーション）。取消後はExcel出力不可。

```
以下の見積・図面アップロード・図面自動拾い（EPIC-04 / EPIC-05 / EPIC-06）のユーザーストーリー・受け入れ基準を、**現在の実装に合わせて**修正してください。実装が「正」です。

【EPIC-04 図面アップロード】

**US-0401（F-01）PDF図面アップロード**
- 受け入れ基準を次のように変更する：
  1. 「Given PDFファイル When アップロード Then 正常に登録され図面一覧に表示される」  
     → 「Given PDFファイル When アップロード Then 正常に登録される（※図面一覧は廃止のため一覧表示はしない）」に変更する。
  2. 「Given PDF以外 When アップロード Then 受付拒否し理由を表示する」→ そのまま。
  3. 「Given 50MB超または200ページ超 When アップロード Then 受付拒否し理由を表示する（暫定NFR）」→ そのまま。
  4. 「Then アップロード後、対象ページ未指定の場合は全ページを対象として自動拾いが自動実行される」  
     → 削除する。または「（未実装：アップロード後の自動拾い実行は行わない。また図面一覧廃止のため、手動で自動拾いを開始するUIも現状ない）」と注記する。
  5. 「Then 図面にステータス（未処理→処理中）が付与される」  
     → 「Then 図面にはステータス（未処理等）が付与される（処理中へは、自動拾い開始UIが現状ないため通常は遷移しない）」のように注記する。
  6. 「Then アップロード操作は監査ログに残る（対象：図面ID）」→ そのまま。

---

【EPIC-05 図面自動拾い】

**実装上の前提**  
図面一覧を廃止したため、「自動拾い開始」「結果」「詳細」「再実行」等のボタンがUIに存在しない。自動拾い・結果表示・編集・再実行の**導線は画面から利用できない**。ロジック（startTakeoff、パイプライン、結果ダイアログ等）はコードに残るが、呼び出し口がない。

**US-0501（F-02）自動拾い開始**
- 受け入れ基準を実装に合わせて修正する：
  - 「図面がアップロードされた Then 全ページを対象に自動拾いが自動実行され…」→ 削除するか「（未実装：アップロード後の自動実行はなし。また図面一覧廃止のため手動開始ボタンもなし）」と注記する。
  - 「Given 既に処理中 Then 二重起動しない」「ジョブIDが発行され…」は、自動拾いが実行される経路が現状ないため、ストーリー全体を「未実装（導線廃止）」と明記するか、該当基準を「（導線廃止のため未使用）」と注記する。

**US-0502〜0505（AI/OCR・失敗時ハンドリング）**
- システム向けストーリーのため、文言はそのまま残してよい。ただし「UI上の設定画面は廃止」（US-0503, 0504）は実装通りなので変更不要。

**US-0506（F-02）拾い結果表示**
- 受け入れ基準を修正する：「結果画面を開く」導線が図面一覧廃止により存在しないため、「（図面一覧廃止のため、結果画面を開くUIは現状ない。処理中/完了/失敗の表示は未使用）」と注記するか、ストーリーを「未実装（導線廃止）」とする。

**US-0507（F-02）拾い結果の編集**
- 同上。結果画面を開く導線がないため、「未実装（導線廃止）」または「（結果画面への導線なし）」と注記する。

**US-0508（F-02）再実行**
- 同上。再実行ボタンは結果ダイアログ内にのみ存在し、結果ダイアログを開く導線がないため、「未実装（導線廃止）」と注記する。

---

【EPIC-06 見積書自動生成】

**US-0601（F-03）拾い結果から見積生成**
- 受け入れ基準を実装に合わせて修正する：
  - 「拾い結果」は、図面の自動拾い結果ではなく**「デモを読み込む」で投入した算出結果（extractedItems）**を指すと明記する。
  - 「算出結果タブで『拾い結果から見積を作成』を押下すると見積（下書き）が作成される」という導線であることを受け入れ基準に反映する。
  - 単価の決定ロジック・AI算出バッジ・税率・端数は実装どおりなので、既存の受け入れ基準でよい。

**US-0602〜0608**
- 見積編集・顧客・案件紐づけ・確定・改版（複製）・取消・Excel出力・一覧・検索は実装あり。受け入れ基準の文言は実装に合わせて微修正する程度でよい。取消理由は必須入力、取消後はExcel出力不可であることを基準に明記する。
```
